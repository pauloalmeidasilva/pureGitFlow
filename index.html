Bem-vindo ao Pure GitFlow!

Este projeto foi desenvolvido pensando em novos programadores que querem saber como grandes empresas trabalham o versionamento de projetos. Nem todas empresas seguem essa rotina, mas aqui está proposta uma rotina mais genérica que pode ser moldada para o padrão da empresa. Consulte seu superior imediato para confirmar isto.

Segue os passos de versionamento: instalação em máquina local, Criação das ramificações necessárias, configuração do arquivo .gitignore para o projeto, commit básico, criação de uma nova funcionalidade, criação de novas realises, criação de novas hotfix.

Instalar Git

Download em git-scm.com
git --version

Configurar .gitignore

echo "node_modules/" >> .gitignore
echo "*.log" >> .gitignore

1. Inicializar

git init
git branch -M main
↓

2. Develop Branch

git checkout -b develop

Commit Diário

git add .
git commit -m "feat: adiciona login"
git push origin develop

Feature Start

git checkout develop
git checkout -b feature/login
↓

Feature Finish

git checkout develop
git merge --no-ff feature/login
git push origin develop

Release Start

git checkout -b release/v1.0.0
↓

Release Finish

git checkout main
git merge --no-ff release/v1.0.0
git tag v1.0.0

Hotfix

git checkout main
git checkout -b hotfix/v1.0.1
git checkout main && git merge --no-ff hotfix/v1.0.1
