Guia Visual Git Flow (Comandos Puros)

Main / Master
Produção
1. Inicialização do Projeto git init git branch -M main git commit -m "Initial commit"
2. Recebe Release ou Hotfix git merge --no-ff release/v1.0 git tag -a v1.0
Hotfix
Correção Crítica
Início (via Main) git checkout main git checkout -b hotfix/bug-fix
↓
Fim (Main + Develop) # Na Main git merge hotfix/bug-fix # Na Develop git merge hotfix/bug-fix
Release
Preparação
Início (via Develop) git checkout develop git checkout -b release/v1.0
↓
Fim (Main + Develop) # Finaliza na Main # e sincroniza Develop
Develop
Integração
Setup inicial git checkout -b develop git push -u origin develop
Recebe Features git merge --no-ff feature/xyz
Feature
Novas Funcionalidades
Início (via Develop) git checkout develop git checkout -b feature/nome
↓
Fim git checkout develop git merge --no-ff feature/nome git branch -d feature/nome

Dica: Use sempre --no-ff para manter o histórico visual de merges.
